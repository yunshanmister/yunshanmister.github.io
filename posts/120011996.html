Maven

maven
常用工具
发布日期:   2020-04-11

Maven是 Apache 下的一个纯 Java 开发的开源项目。Maven 是一个项目管理工具，可以对 Java 项目进行构建、依赖管理。

官网

Maven 功能

Maven 能够帮助开发者完成以下工作：

  • 构建
  • 文档生成
  • 报告
  • 依赖
  • SCMs
  • 发布
  • 分发
  • 邮件列表

Maven具有的特点

优点

1.简化项目的依赖管理

2.有助于多模块，分布式的项目开发

3.Maven具有很多的插件，例如tomcat,自动发布版本等

4.简单易于上手。

5.十分方便项目的维护与发布

缺点

1.国内网络不适合从官方仓库下载，要配置镜像。

2.需要下载，导入，比较慢，容易出现错误。

3.没有一个具体的文档描述。

Maven 有 Snapshot 版本与 Release 版本

  • Snapshot 版本代表不稳定、尚处于开发中的版本。

  • Release 版本则代表稳定的版本。

Maven版本要求

Maven 3.3 要求 JDK 1.7 或以上
Maven 3.2 要求 JDK 1.6 或以上
Maven 3.0/3.1 要求 JDK 1.5 或以上

版本 要求
Maven 3.1 JDK 1.5 或以上
Maven 3.2 JDK 1.6 或以上
Maven 3.3 JDK 1.7 或以上

Maven下载

下载完成解压配置环境变量

Windows系统

右键 “我的电脑“，选择 “属性“，之后点击 “高级系统设置“，点击”环境变量“，设置”系统变量“。

1.新建系统变量 MAVEN_HOME，变量值：Maven解压路径,如:E:\Maven\apache-maven-3.6.3

2.编辑系统变量 PATH，添加变量值：;%MAVEN_HOME%\bin

然后一路确定就配置完成

Linux系统
可以自己在Linux系统上面下载或者上传然后解压，完成后编辑 /etc/profile 文件。
命令 sudo vim /etc/profile
在文件末尾添加如下代码

export MAVEN_HOME=/usr/local/apache-maven-3.6.3     
export PATH=${PATH}:${MAVEN_HOME}/bin

保存文件后使用 # source /etc/profile 命令使环境变量生效

所有配置完成输入 # mvn -v 出现版本信息说明安装配置结束。

Maven仓库

1.本地仓库
Maven 的本地仓库，在安装 Maven 后并不会创建，它是在第一次执行 maven 命令的时候才被创建。运行 Maven 的时候，Maven 所需要的任何构件都是直接从本地仓库获取的。如果本地仓库没有，它会首先尝试从远程仓库下载构件至本地仓库，然后再使用本地仓库的构件。

默认位置在： 用户目录下一个路径名为 .m2/respository/ 的目录。

配置自己的路径：在maven目录下的conf目录里面有个settings.xml的配置文件在里面配置

E:/repository

2.中央仓库 [http://search.maven.org/#browse]
Maven 中央仓库是由 Maven 社区提供的仓库，其中包含了大量常用的库。这个仓库由 Maven 社区管理。不需要配置。需要通过网络才能访问。

3.远程仓库
是开发人员自己定制仓库，包含了所需要的代码库或者其他工程中用到的 jar 文件。

Maven常用插件

以下是我经常用到的插件

插件 作用
clean 构建之后清理目标文件。删除目标目录
compiler 编译 Java 源文件
surefile 运行 JUnit 单元测试。创建测试报告
jar 从当前工程中构建 JAR 文件
war 从当前工程中构建 WAR 文件
javadoc 为工程生成 Javadoc
tomcat7 tomcat
